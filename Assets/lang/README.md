# Translations

This folder holds the interface translations. Spanish and English are built
into the app, so there are no files for them here. Every other language is a
single JSON file named with its language code, for example `it.json` for
Italian or `fr.json` for French.

Each file maps an English interface string to its translation:

    {
      "Settings": "Impostazioni",
      "Start server": "Avvia server"
    }

The English string on the left is the key. The app looks each one up while it
runs. If a key is missing, or its value is left empty, the app shows the
English text instead, so an unfinished translation never breaks the interface.

## Add a new language

1. Copy `_template.json` to a file named after your language code, for example
   `de.json` for German. The template lists every translatable string with an
   empty value.
2. Fill in the translations. You can translate as many or as few strings as you
   want; anything you leave empty stays in English.
3. Update the status table and open a pull request:

       python3 scripts/translation-status.py

4. Build the app with `./make-app.sh`. Your language appears in
   Settings > Language on its own.

That is all that is needed. You do not have to touch any Swift code.

## How pull requests are checked

Every pull request that touches a translation file runs an automatic check. It
fails if a file has broken JSON or a key that is not in the template, or if the
status table above was not regenerated. So before pushing, run:

    python3 scripts/check-translations.py
    python3 scripts/translation-status.py

If you would rather have this happen on every commit automatically, the
repository also ships an optional Git hook. Enable it once in your clone:

    git config core.hooksPath scripts/hooks

## Update an existing translation

Open the language file and edit the values. To add strings that were introduced
in a newer version of the app, regenerate the template first:

    python3 scripts/extract-strings.py

This rewrites `_template.json` with the current list of strings. Compare it with
your language file and add whatever is missing.

## Notes

- Do not change the keys (the English text on the left). They must match the
  app's text exactly, or the lookup will not find them.
- A few strings that contain values inserted at runtime (file names, numbers,
  and similar) are not in the template. Those always stay in English.
- `_template.json` is only a reference for translators. It is not shipped with
  the app.
- The same files also translate the standalone web console served by
  llama-server, so one translation covers both the app and the web chat.

## Contribute it back

If you would like your translation included in the project, open a pull request
that adds your language file. Improvements to existing translations are welcome
too.
